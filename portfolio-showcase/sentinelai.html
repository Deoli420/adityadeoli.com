Motivation

The Problem I Solved

Blind spots in production. Teams ship APIs but rely on users to report outages. By the time a 500 error hits a Slack channel, hundreds of requests have already failed silently.

No risk context. Existing uptime monitors say "it's down" but they don't quantify how risky an endpoint has become over time. A slow degradation is invisible until it cascades.

Security as an afterthought. Most monitoring dashboards lack tenant isolation, proper auth, and SSRF protection. Internal tools become attack vectors.

Core Capabilities

What SentinelAI Does

Every feature designed, architected, and implemented by me — from database schema to pixel-perfect UI.

Real-Time Monitoring

Automated health probes hit your endpoints on a configurable schedule. Tracks response times, status codes, SSL validity, and DNS resolution from a single dashboard with auto-refresh.

Anomaly Detection

Statistical analysis flags abnormal latency spikes, unexpected status codes, and response deviations. Every run is scored against a rolling baseline so you catch regressions before users do.

Risk Scoring Engine

Each endpoint gets a composite risk score (0–100) computed from latency percentiles, failure rates, anomaly frequency, and cert expiry. Four severity tiers: LOW → MEDIUM → HIGH → CRITICAL.

Intelligent Alerting

Configurable alert rules powered by n8n webhook integrations. Set thresholds per-endpoint, define escalation chains, and get notified through Slack, email, or any webhook service.

Multi-Tenant Auth & RBAC

JWT access tokens with httpOnly refresh-token rotation. Organization-level data isolation with three roles — Admin, Member, Viewer — enforced at both API and UI layers.

Server-Side Proxy

Test any API directly from the dashboard without CORS headaches. Built-in SSRF protection blocks private networks, cloud metadata, and localhost. Supports all HTTP methods, auth types, and body formats.

Tradeoffs

Key Engineering Decisions

The "why" behind the architecture — real tradeoffs, not textbook answers.

Same-Origin Serving over Separate Frontend Deploy

nginx serves both the React SPA and proxies /api/ on the same domain (sentinelai.adityadeoli.com). This eliminates CORS complexity entirely and lets httpOnly cookies work with SameSite=Strict — the most secure cookie configuration available.

Denormalized organization_id on api_runs

ApiRun stores organization_id even though it could be inferred via the parent endpoint. This denormalization lets me filter runs by tenant without joining through api_endpoints on every dashboard query — critical for keeping the stats endpoint fast.

Refresh Token Rotation over Long-Lived Access Tokens

Access tokens expire in 15 minutes. On 401, the frontend silently calls /auth/refresh which atomically rotates the refresh token (old one revoked, new one issued). Stolen tokens have a blast radius of minutes, not days.

Statistical Baselines over Fixed Thresholds

Rather than alerting on "latency > 1000ms", the anomaly detector compares each response against a rolling average. A 200ms endpoint spiking to 800ms is far more concerning than a 2s endpoint at 2.2s — context matters.

Repository Pattern over Raw SQL

Every database operation goes through a repository class with explicit tenant_id parameters. This makes tenant-scoping auditable (grep one directory), testable (mock the repo), and migration-safe (schema changes in one place).

Reflections

What I Learned Building This

Production Auth is Hard

Token rotation, race conditions on concurrent 401 retries, httpOnly cookie paths, lockout mechanics — each has subtle failure modes that only surface under real load. I built a request-queuing interceptor that silently refreshes and replays failed requests.

Multi-Tenancy Touches Everything

Adding organization_id isn't just a column — it changes every query, every service method, every API response. I had to systematically audit all 3 layers (routes → services → repositories) for runs, anomalies, and risk scores to eliminate data leakage.

nginx Header Inheritance is Brutal

Any add_header in a child location block silently drops ALL parent headers. I had to repeat every security header in every location context — a footgun that cost me hours of debugging missing CSP headers in production.

SSRF is a Real Threat

A proxy endpoint that lets users hit any URL is an SSRF vector. I added hostname resolution → IP check against RFC 1918 ranges, carrier-grade NAT, IPv6 private ranges, cloud metadata IPs, and a blocked-hostname list.
